正则表达式验证手机号、固话

发表于 2020-03-17

规则

固话

区号是以 0 开头的，后面是 2-3 位，正则表达式0\d{2,3}。固定电话号码由 7~8 位数字，则使用表达式\d{7,8}来进行匹配。固话的组合方式可能是“区号-号码”或是“区号号码”

bash /0\\d{2,3}[-]?\\d{7,8}|0\\d{2,3}\\s?\\d{7,8}/ bash

手机号

手机号是 11 位数，并且以数字 1 开头

bash /^[1](([3][0-9])|([4][5-9])|([5][0-3,5-9])|([6][5,6])|([7][0-8])|([8][0-9])|([9][1,8,9]))[0-9]{8}$/ bash

示例

``` bash
function verifyMobile($mobile)
{
$mobileRule = “/^1[0-9]{8}$/“;
$telRule = “/0\d{2,3}[-]?\d{7,8}|0\d{2,3}\s?\d{7,8}|13[0-9]\d{8}|15[1089]\d{8}/“;

    if (!preg_match($mobileRule, $mobile) && !preg_match($telRule, $mobile)) {
        return false;
    }

    return true;
}

$isVerifyPass = verifyMobile('1856562522'); 
var_dump($isVerifyPass);

``` bash

网络协议-网络层

发表于 2019-11-19

网络层

主要是终端节点之间的通信，将网络包从源主机发送到目标主机，从而实现点对点的通信；
可以跨越不同的数据链路，从而实现两节点之间数据包的传输，不同数据包之间的传输是通过路由器连接的；

局域网内通过数据链路层实现通信，而局域网之间则需要通过网络层来实现数据包的传输；

IP地址

IP地址是TCP/IP协议的基石；主要用于在连接到网络的所有主机中识别出进行通信的目标地址【在TCP/IP中所有主机或路由器都需要设定IP】；

MAC地址用于标识统一链路上不同的计算机，然后网络层则通过IP地址标识地址信息；

格式

分为 IPv4和IPv6；
IPv4是由32位二进制表示的，按照8位一组的方式分为4组，将每组的二进制转换为十进制，就是我们常见的IP地址了 

如： 192.168.1.125  [私有IP]

组成

通常由网络标识和主机标识组成；

网络标识【网络地址】用于标识节点所属的数据链路，不同局域网之间网络地址必须不同，同一局域网内的主机必须有相同往来地址；
主机标识【主机地址】用于标识同一局域网内的不同主机；

192.168.1.125  192.168.1为网络地址；125为主机地址；

分类

A类 以0开头，1~8位为网络地址，后24位为主机地址；
B类 以10开头，1~16位为网络地址，后16位为主机地址；
C类 以110开头，1~24位为网络地址，后8位为主机地址；
D类 以1110开头，1~32位为网络地址，无主机地址；

子网掩码

子网掩码是由32位二进制表示，格式同IP地址；

IPv6

出现的目的是从根本上解决IPv4不够用的问题；

广播地址和回环地址

广播地址用于在同一局域网中相互连接的主机间发送数据【IP地址主机地址部分设置为1，就是广播地址】；

回环地址同一计算机内部的程序直接进行网络通信时使用的默认地址，一般使用127.0.0.1【localhost】作为回环地址，使用回环地址数据包不会流向网络；

这种方式没有授权码步骤，所以称为隐藏式；

第一步：A 网站提供一个链接，要求用户跳转到 B 网站，授权用户数据给 A 网站使用。

上面 URL 中，response_type参数为token，表示要求直接返回令牌。

第二步：用户跳转到 B 网站，登录后同意给予 A 网站授权。这时，B 网站就会跳回redirect_uri参数指定的跳转网址，并且把令牌作为 URL 参数，传给 A 网站。

上面 URL 中，token参数就是令牌，A 网站因此直接在前端拿到令牌。

注意：令牌的位置是 URL 锚点（fragment），而不是查询字符串（querystring），这是因为 OAuth 2.0 允许跳转网址是 HTTP 协议，因此存在”中间人攻击”的风险，而浏览器跳转时，锚点不会发到服务器，就减少了泄漏令牌的风险。

这种方式把令牌直接传给前端很不安全的。所以隐藏式只用于一些安全要求不高的场景，并且令牌的有效期必须非常短，
通常就是会话期间（session）有效，浏览器关掉，令牌就失效了。

OAuth2.0

发表于 2019-09-03

OAuth是一种授权机制。数据所有者同意第三方应用进入系统，获取数据。系统就会产生一个短期的令牌供第三方应用使用。

作用

授权第三方应用，获取用户的数据。

优点

让第三方可以获取到数据，又可以随时撤销。

令牌与密码的区别

$ 令牌是短期的，到了失效时间会自动失效，不能修改；密码如果用户自己不修改，就不会变化；
$ 令牌可以被用户主动的撤销，撤销后不可再使用；密码不能被撤销；
$ 令牌权限有限，比如进入一个院子，需要先输入大门的令牌，进入房间内还需要另外的令牌，一般只读令牌的权限小于读写令牌的权限；密码一般都是完整权限；

获取令牌方式

获取令牌的方式有四种
$ 授权码
$ 隐藏式
$ 密码式
$ 客户端凭证

无论哪一种方式，第三方应用在申请令牌前，都必须到系统备案，说明自己是身份，然后拿到两个身份识别码：客户端ID和客户端密钥；
这样做的目的是为了防止令牌滥用，不备案就不能拿到身份识别码。

令牌使用

A 网站在拿到令牌以后，就可以通过API向 B 网站请求数据了，每次请求的API都必须带令牌
通过在请求的头信息，增加一个 Authorization 字段

令牌更新

令牌到期后，如果需要用户重新将上述流程重复一遍的话，体验就不怎么好了，OAuth2.0允许用户自动更新令牌；

B 网站颁发令牌的时候，一次性颁发两个令牌，一个用于获取数据，另一个用于获取新的令牌（refresh token 字段）。令牌到期前，用户使用 refresh token 发一个请求，去更新令牌。

上面 URL 中，grant_type参数为refresh_token表示要求更新令牌，client_id参数和client_secret参数用于确认身份，refresh_token参数就是用于更新令牌的令牌；
B 网站验证通过以后，就会颁发新的令牌。
